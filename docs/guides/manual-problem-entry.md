# 수동 문제 업로드 가이드

Phase 2부터 `/teacher/problems/new` 폼은 **외부 LLM(ChatGPT / Gemini / Claude 웹)** 으로 문제 이미지를 텍스트로 변환한 뒤 사용자가 직접 붙여넣는 방식이 주 경로다. Anthropic API 비용 없이 기존 LLM 구독을 활용한다.

이 문서는 **한 문제를 준비해 저장하기까지의 전체 절차**와, **외부 LLM에 그대로 붙여넣을 프롬프트 템플릿**을 제공한다.

---

## 0. 사전 준비

- Teacher 계정으로 로그인 (`https://math-assistant-flame.vercel.app/login`)
- 문제 이미지 1장을 파일로 저장 (교재 스캔, 스크린샷 등)
- 외부 LLM 탭 열기 (Claude 웹 Opus 권장 — 한국어 수식 이해도가 가장 높음)

---

## 1. 외부 LLM에 붙여넣을 프롬프트 템플릿

아래 블록 전체를 Claude / ChatGPT / Gemini에 **이미지와 함께** 붙여넣는다. 프롬프트는 한국어 수학 문제에 최적화되어 있고, 출력 포맷은 앱 폼의 4개 필드와 1:1로 매칭된다.

````text
너는 한국 중학교/고등학교 수학 교재의 문제를 구조화된 형식으로 추출하는 도우미다.

첨부한 이미지에서 **단 한 개의 문제**만 추출해, 아래 네 개의 블록으로 나눠서 출력해라.
다른 설명, 주석, 머리말, 마크다운 코드 펜스(```)는 포함하지 말 것. 네 개의 `===` 블록만 출력.

===문제 본문===
(문제 전문을 그대로 옮겨라. 다음 규칙을 지킬 것:
- 한국어는 한국어 그대로
- 수식은 LaTeX로 변환: 인라인은 $...$, 디스플레이는 $$...$$
  예: x² → $x^2$, √2 → $\sqrt{2}$, 분수 1/2 → $\frac{1}{2}$
- 그림/도형/그래프는 문제 끝에 "(그림 생략)" 한 줄 추가
- 보기 ①②③④⑤는 줄바꿈으로 구분해 모두 포함
- 문제 번호는 제거, 본문만 남김)

===정답===
(이미지에 정답이 있으면 그대로 옮긴다. 수식은 동일하게 $...$ 규칙.
정답이 없거나 불확실하면 이 블록에 "없음"이라고만 적어라.)

===난이도===
(1~5 중 한 숫자. 추정 기준:
1 = 교과 기본 (개념 직접 적용)
2 = 약간 응용
3 = 표준 연습 (대부분의 교재 본문 수준)
4 = 상위 응용 (내신 심화 수준)
5 = 킬러 (수능/경시 난이도)
모르겠으면 "3"으로 적어라.)

===태그===
(쉼표로 구분한 2~5개의 주제 키워드. 중3 과정 기준으로 구체적으로.
예: 이차방정식, 근의 공식
예: 인수분해, 곱셈 공식
예: 제곱근, 분모의 유리화)
````

## 2. LLM 출력 해석

LLM이 위 프롬프트에 따라 아래와 같은 형식으로 답한다:

```text
===문제 본문===
다음 부등식을 만족시키는 자연수 $x$의 개수를 구하시오.
$$2 < \sqrt{x} < 3$$

===정답===
없음

===난이도===
2

===태그===
제곱근, 부등식, 자연수
```

각 블록이 앱 폼의 어떤 필드에 들어가는지는 다음과 같다:

| LLM 블록 | 앱 폼 필드 | 필수 여부 |
|----------|-----------|-----------|
| `===문제 본문===` | **문제 본문** textarea | ✅ 필수 |
| `===정답===` | **정답** input | 선택 (값이 "없음"이면 비워둠) |
| `===난이도===` | **난이도** select (1~5) | 선택 (값이 "3"이어도 선택 가능) |
| `===태그===` | **태그** input (쉼표 구분) | 선택 |

---

## 3. 앱 폼에 저장

1. `/teacher/problems/new` 접속
2. **대단원** 선택 (예: 제곱근과 실수)
3. **유형** 선택 (예: 제곱근의 뜻과 성질)
4. **문제 본문**에 `===문제 본문===` 블록의 내용만 복사해 붙여넣기 (구분자 `===...===`는 **포함하지 말 것**)
5. **정답** 입력 (LLM이 "없음"이라 했으면 비워둠)
6. **난이도** 드롭다운에서 LLM이 제시한 숫자 선택
7. **태그** 입력 (쉼표 구분 그대로 복붙)
8. **원본 이미지** (선택) — 교재 이미지 첨부하면 원본 보관됨. 없어도 저장 가능
9. **저장** 버튼 클릭
10. 성공 시 "문제 상세 보기" 링크를 눌러 LaTeX 수식이 제대로 렌더되는지 확인

---

## 4. 팁과 주의사항

### LaTeX 수식 오류 발견 시
- 문제 상세 화면에서 수식이 깨져 보이면, 수정 페이지는 아직 없으므로 해당 문제를 삭제(추후 Phase에서 추가 예정)하거나 재저장.
- 자주 틀리는 패턴:
  - ❌ `x^2` (달러 밖) → ✅ `$x^2$`
  - ❌ `\sqrt2` (중괄호 없음) → ✅ `$\sqrt{2}$`
  - ❌ `1/2` (수식 밖) → ✅ `$\frac{1}{2}$`

### 한 이미지에 여러 문제가 있을 때
- 폼은 **1 문제 = 1 저장**이 원칙. 여러 문제를 이미지 한 장에 뽑았으면 LLM에 "위 이미지에서 2번 문제만" 같이 지정해서 다시 질문.
- 이미지를 미리 crop해서 한 문제씩 저장하는 것도 좋음.

### 도형 · 그래프 문제
- Phase 2에서는 도형 자체를 앱에 저장하지 않는다. 문제 본문에 `(그림 생략)` 주석만 남기고, 원본 이미지를 **원본 이미지 필드**에 첨부해 보관.
- 문제 상세 화면에서 원본 이미지를 참조할 수 있다.

### 태그 예시 (중3-1 범위)

| 단원 | 태그 예 |
|------|---------|
| 제곱근과 실수 | `제곱근`, `무리수`, `분모의 유리화`, `실수의 대소` |
| 다항식의 곱셈과 인수분해 | `곱셈 공식`, `인수분해`, `완전제곱식`, `치환` |
| 이차방정식 | `이차방정식`, `근의 공식`, `판별식`, `근과 계수의 관계` |
| 이차함수 | `이차함수`, `꼭짓점`, `축의 방정식`, `최댓값과 최솟값` |

## 5. 변형 문제 생성 (현재 동작 조건)

문제 상세 화면의 **변형 생성** 버튼은 현재 `ANTHROPIC_API_KEY`가 Vercel production에 설정되어 있어야 동작한다. 키가 없으면 에러 메시지가 뜬다.

세 가지 선택지:
- **A**: Vercel Settings → Environment Variables 에 `ANTHROPIC_API_KEY` 추가 후 재배포 → 버튼으로 자동 생성 (API 비용 발생)
- **B**: 변형도 수동으로 — 외부 LLM에 "위 문제의 수치만 바꾼 변형을 JSON 없이 같은 형식으로 생성해" 프롬프트 후 같은 `/teacher/problems/new`에 다시 저장 (원본과 별도 row이지만, 지금 스키마에는 variant 관계가 없어서 "다른 문제"로 저장됨 — 변형 관계 기록이 필요하면 A가 맞음)
- **C**: 현 상태 유지, 변형 기능은 사용 안 함

이 결정은 Phase 2 마무리 시점에 ADR로 기록 예정.
